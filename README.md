# Whitepace SaaS Landing Page

A simple, clean landing page built for Whitepace. This web page displays an interactive layout which explains what the page is for, projects, tailored options, and subscription tiers, amongst others.

## 📌 Problem Statement

- Provides digital views and easy access to what this page is all about. Right from the first glance, you find an actionable button to click.
- The various grid sections break down information that could take long to explain to a client in person into a chronological, detailed layout order.
- This page helps users visualize the pricing services offered by Whitepace, including the free entry-level services.
- It goes further to give testimonials from people who have used Whitepace before, ensuring the landing page builds immediate trust.

## 🎯 Project Goals

- Convince more people to sign up for Whitepace.
- Explain what Whitepace is all about in seconds.
- Ensure the page can be accessed easily from any electronic device.

## 🛠 Tech Stack

- Frontend: Vanilla HTML5 and CSS3
- Icons: FontAwesome v6

## 🖥 Features

- Fully Responsive: Optimized design layout for desktops, tablets, and mobile devices.
- Modern Layouts: Fluid layout grids and flexbox systems built entirely using clean CSS properties.
- Visual Illustrations: Decorative backgrounds and high-quality images for easy illustration and visualization.
- Pricing Strategy: Clear pricing layout using an emphasized subscription card matrix section.
- Typography Hierarchy: Styled using a clear font-size hierarchy, making the site look professional and user-friendly.

## 🧠 Challenges Faced

- Asset Sourcing: Navigating how to obtain and format the correct images used throughout the landing page design.
- Spacing and Layouts: Managing the precise usage of padding, margins, and grid gaps for each individual feature section.
- Positioning: Solving layout bugs related to relative and absolute CSS positioning when placing the decorative circle assets.
- Aspect Ratios: Balancing aspect ratios, scaling, and flex-wrapping behaviors, especially within the application grids.
- Footer Complexity: Structuring the footer layout cleanly, as the nested columns proved difficult to align at first.
- Mobile Responsiveness: Managing the media query transitions was the most challenging and critical part of the build.

## 📚 What I Learned

- How to leverage advanced CSS Grid and Flexbox alignment rules.
- Implementing semantic, SEO-friendly HTML5 architecture.
- How to cleanly insert vector graphics, images, links, and FontAwesome icons.
- Setting background images directly within localized CSS classes.
- Structuring reusable, interactive action buttons.
- Using CSS overflow configurations to eliminate layout breaking scrollbars.
- Practical application of span, ul-li structures, z-index layering, and fluid typography.
